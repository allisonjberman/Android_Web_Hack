General Information

This project was created for Software Testing during the Spring 2017 semester at the University of Texas at Austin.

The purpose of this project is to test and debug an Android phone. This project has two aims:

  • Obtain continuous diagnostic information from an Android phone
  • Run tests on the phone to simulate desired testing conditions, such as low battery

    • Aim 1: Continuous Android Diagnosis

    We report information from the Android phone overtime including: battey level, battery temperature, battery status, CPU usage, and RAM. This can be used to confirm or help diagnose problems users may experience with their phones. For example, if a user compains that her phone is running slowly, or the battery is draining quickly, she can see how the battery level is changing over time and which apps are the most CPU intensive.

    Aim 2: Test Android Conditions

    It is good practice to test software in extreme conditions, but it is generally undesirable to actually cause these conditions. We can mimic these types of situations (low battery, high temerature, etc.). We test that certain conditions cause different behavior for our program. For example, in low battery situations we send debug data less frequently and we test that it acutally does. We display whether these tests pass/fail for our application.

    Our Methods

    We used the command "adb dumpsys" to obtain diagnostic information from the Android phone. To continuously and wirelessly send debug information, we rooted the Android and pushed a Python script that captures relevant data. This diagnostic information is displayed in Statistics and updates every 5 minutes.


    When running custom tests on the phone, the user selects which test to run on the webapp. This sends a post request to the Android phone to run the relevant Python script. Pass/fail results are displayed in Run Tests.
